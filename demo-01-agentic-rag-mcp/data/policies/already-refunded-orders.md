# Already-Refunded Orders

This policy covers the situation where a refund has already been issued against an order and the customer requests another refund for the same order.

## Core Rule: No Second Refunds

Once a refund has been successfully issued for an order, **no second refund may be issued for that same order**. This applies whether the prior refund was full or partial. Issuing a duplicate refund results in over-payment to the customer and must be avoided.

## How to Identify an Existing Refund

Before processing any refund, check the order for an existing refund reference number. An order that already shows a recorded refund — full or partial — has been refunded and is not eligible for another.

## What to Tell the Customer

When a customer requests a refund for an order that was already refunded:

- Confirm politely that a refund has already been processed for the order.
- Provide the **existing refund reference number** so the customer can locate it.
- State the amount that was refunded and the date it was issued.
- Explain that the refund returns to the original payment method and may take time to appear, depending on the payment provider.

## Exceptions

If the prior refund was only partial and the customer is now entitled to the remaining balance under another policy, that remaining amount may be considered — but it is treated as a new, separate adjustment and must never duplicate the amount already refunded. Combined refunds may never exceed the original order amount.

If the customer disputes that they received the prior refund, escalate so the disbursement can be traced. Do not issue a second refund to resolve a tracing question.
